## How to use line references

Every line reference in this repo (e.g. "line 33", "lines 660–688") indexes into the file at the CL4R1T4S URL above. To follow a reference:

1. Open the raw text URL.
2. In your editor / terminal, jump to the cited line number.
3. The reference resolves to that exact byte range in the mirror.

If the author of CL4R1T4S later edits the file, line numbers in this repo may drift. If that happens, the commit hash pinned below identifies the version this analysis was written against.

Subsequent edits to the CL4R1T4S file may shift line numbers; open an issue on this repo if you find a line reference that no longer resolves to the expected content, and we'll either update the reference or (preferred) pin a specific CL4R1T4S commit hash here.

## Why not host the file

Three reasons:

1. **Hosting the original is already done.** CL4R1T4S has the file, an issue tracker, and a maintainer. A second copy adds nothing and fragments feedback.
2. **DMCA surface.** This repo is research analysis, not a re-host. Keeping the verbatim file out of the repo makes the research/commentary posture of the repo unambiguous.
3. **Staleness.** If Anthropic updates the prompt and CL4R1T4S mirrors the update, the canonical mirror is more current than a pinned copy here.

If the CL4R1T4S mirror is ever taken down and there's no successor archive, we'll revisit this decision.

## What this repo contributes

Not the text — the **analysis**. The twelve primitives, seven techniques, five patterns, ten section annotations, and the template with a worked example are the added value. Line references point you back to the source for verification.
